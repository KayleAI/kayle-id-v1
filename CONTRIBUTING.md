# Contributing to Kayle ID

Thanks for your interest in contributing to Kayle ID.

Before opening a PR:

- search [existing issues](https://github.com/KayleAI/kayle-id/issues) and [pull requests](https://github.com/KayleAI/kayle-id/pulls) so we don't end up with duplicate work
- if your change touches AI-assisted code, also read [AGENTS.md](./AGENTS.md) — it captures the architectural rules every contribution is expected to follow

## Signed commits

The default branch enforces signed commits. Configure Git with a GPG, SSH, or S/MIME key that's registered with your GitHub account before you push, otherwise the branch protection rule will reject the merge. See [GitHub's signing-commits guide](https://docs.github.com/en/authentication/managing-commit-signature-verification/signing-commits).

A `Signed-off-by` trailer is **not** sufficient — commits must show as `Verified` on GitHub.
