# Security Policy

## Reporting a vulnerability

If you believe you have found a security vulnerability in Kayle ID, email
`security@kayle.id` with the details.

Do not create a public issue, discussion, or pull request for a security
vulnerability or a proposed vulnerability fix. Email us first so we can triage
the report and coordinate remediation privately.

Useful reports include:

- a clear description of the vulnerability and its impact
- steps to reproduce or a proof of concept
- the affected Kayle ID surface, version, or commit SHA
- any relevant logs, screenshots, or requests with secrets and personal data
  redacted
- the best way to contact you for follow-up questions

## Disclosure

Please do not publicly disclose the issue until we have investigated it and
coordinated a fix or mitigation. We will work with reporters in good faith and
will credit responsible disclosures when appropriate and requested.
